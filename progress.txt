## Codebase Patterns
- To avoid circular dependencies, shared functions like `refreshAndPush` go in `usageService.ts` (not `index.ts`), since both `tray.ts` and `index.ts` import from it
- Preload script at `src/main/preload.ts` exposes `window.prominenceAPI` for renderer IPC — always use `contextBridge.exposeInMainWorld`
- IPC handlers registered in `src/main/index.ts` inside `app.whenReady().then()` block
- Main process TypeScript files go in `src/main/`, renderer HTML/CSS in `src/renderer/`
- Use `nativeImage.createFromBuffer()` for programmatic tray icons (no external PNG needed)
- Electron `window-all-closed` handler must not call `app.quit()` for tray apps
- TypeScript compiles from `src/` to `dist/`, entry point is `dist/main/index.js`
- For transparent frameless windows: `html,body { background: transparent }`, visible background on a container div
- Overlay window is hidden/shown (not destroyed/recreated) for performance — use `getOverlayWindow()` to access it
- Usage data comes from `https://api.anthropic.com/api/oauth/usage` with OAuth token from `~/.claude/.credentials.json` and `anthropic-beta: oauth-2025-04-20` header
- The `claude usage` CLI command hangs when called from child processes — use direct HTTPS API calls instead
- API returns utilization percentages (0-100), not raw counts. Fields: `five_hour`, `seven_day`, `seven_day_opus`, `seven_day_sonnet`, `seven_day_cowork`, `extra_usage`
